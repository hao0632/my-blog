<h2 id="核心技术详解">核心技术详解</h2>
<h3 id="1-前端框架-next-js-14">1. 前端框架：Next.js 14</h3>
<p>博客基于 <strong>Next.js 14</strong> 构建，采用 <strong>App Router</strong> 模式。</p>
<p><strong>主要特点：</strong></p>
<ul>
<li><strong>静态站点生成（SSG）</strong>: 通过 <code>output: 'export'</code> 配置实现纯静态导出</li>
<li><strong>文件系统路由</strong>: <code>src/app</code> 目录下的文件自动映射为路由</li>
<li><strong>React Server Components</strong>: 默认支持服务端组件，提升性能</li>
</ul>
<p><strong>关键配置（next.config.js）：</strong></p>
<pre><code class="hljs language-javascript"><span class="hljs-keyword">const</span> nextConfig = {
  <span class="hljs-attr">output</span>: <span class="hljs-string">'export'</span>,
  <span class="hljs-attr">basePath</span>: process.<span class="hljs-property">env</span>.<span class="hljs-property">NEXT_PUBLIC_BASE_PATH</span> || <span class="hljs-string">''</span>,
  <span class="hljs-attr">images</span>: {
    <span class="hljs-attr">unoptimized</span>: <span class="hljs-literal">true</span>,
  },
}
</code></pre>
<h3 id="2-样式方案-tailwindcss-3">2. 样式方案：TailwindCSS 3</h3>
<p>使用 <strong>TailwindCSS 3.4.1</strong> 作为样式框架，配合 <code>@tailwindcss/typography</code> 插件优化 Markdown 内容展示。</p>
<p><strong>自定义配置（tailwind.config.ts）：</strong></p>
<ul>
<li>自定义主题色：<code>primary: #2563eb</code>（蓝色）、<code>secondary: #7c3aed</code>（紫色）</li>
<li>扩展 typography 样式，移除默认的代码块背景色，便于自定义代码高亮</li>
</ul>
<h3 id="3-markdown-处理流程">3. Markdown 处理流程</h3>
<p>博客使用 <strong>remark</strong> 和 <strong>rehype</strong> 系列工具链处理 Markdown 内容：</p>
<pre><code>Markdown 文件
    ↓
gray-matter (解析 frontmatter)
    ↓
remark (解析 Markdown → AST)
    ↓
remark-gfm (支持 GitHub Flavored Markdown)
    ↓
remark-rehype (转换为 rehype AST)
    ↓
rehype-highlight (代码语法高亮)
    ↓
rehype-raw (保留原始 HTML)
    ↓
rehype-stringify (转换为 HTML 字符串)
</code></pre>
<h3 id="4-代码高亮-highlight-js">4. 代码高亮：highlight.js</h3>
<p>使用 <code>highlight.js</code> 配合 <code>rehype-highlight</code> 实现代码块语法高亮，支持多种编程语言。</p>
<h3 id="5-部署方案">5. 部署方案</h3>
<p>采用 <strong>GitHub Pages</strong> 作为托管平台：</p>
<p><strong>部署流程：</strong></p>
<ol>
<li><code>npm run build</code> - 构建静态站点到 <code>out</code> 目录</li>
<li><code>gh-pages -d out</code> - 将构建产物推送到 <code>gh-pages</code> 分支</li>
</ol>
<p><strong>自动化部署：</strong>
通过 <code>.github/workflows/deploy.yml</code> 配置 GitHub Actions，实现提交代码自动部署。</p>

<h2 id="技术选型理由">技术选型理由</h2>
<h3 id="为什么选择-next-js">为什么选择 Next.js？</h3>
<ol>
<li><strong>静态导出能力</strong>: 适合博客这种以内容为主的站点</li>
<li><strong>文件系统路由</strong>: 简化路由管理，文章文件即为路由</li>
<li><strong>社区成熟</strong>: 丰富的生态系统和文档支持</li>
<li><strong>性能优秀</strong>: 内置优化，无需额外配置</li>
</ol>
<h3 id="为什么选择-tailwindcss">为什么选择 TailwindCSS？</h3>
<ol>
<li><strong>快速开发</strong>: 原子化 CSS 提高开发效率</li>
<li><strong>响应式设计</strong>: 内置响应式断点</li>
<li><strong>主题定制</strong>: 灵活的主题配置能力</li>
</ol>
<h3 id="为什么选择-remark-rehype-生态">为什么选择 remark/rehype 生态？</h3>
<ol>
<li><strong>可扩展性强</strong>: 支持丰富的插件系统</li>
<li><strong>社区活跃</strong>: 持续维护和更新</li>
<li><strong>灵活性高</strong>: 可以定制 Markdown 解析和转换流程</li>
</ol>
